Symfony Blog

A complete blog platform built with Symfony 7, featuring an advanced administration system, full CRUD management for articles, categories and tags, and a comment system with moderation. Deployed with Docker for a reproducible and scalable development environment.

Features

  • Complete CRUD system for articles, categories and tags
  • Admin dashboard with statistics
  • Comment management with moderation system
  • Dynamic filtering by categories and tags
  • Secure authentication system
  • Responsive design with SASS/SCSS

Development Specs

  • Backend: Symfony 7, PHP 8.2+
  • Database: MySQL/MariaDB, MongoDB
  • Frontend: Twig, SASS/SCSS, JavaScript (ES6)
  • Infrastructure: Docker, Docker Compose
  • Testing: PHPUnit
  • Security: Symfony Security Bundle, CSRF Protection

The Challenge

The main challenge of this project was creating a robust and scalable content management system while maintaining a clear separation between the public interface and administration. I had to implement a granular permission system to secure access to different dashboard sections.

Managing relationships between entities (Articles, Categories, Tags, Comments) with Doctrine ORM required particular attention to optimize queries and avoid the N+1 problem. I used custom joins in repositories to efficiently load associated data.

Docker integration also presented interesting challenges, particularly for volume synchronization and database migration management. I created a Makefile to automate common tasks (build, migrations, fixtures) and facilitate onboarding for new developers.

The dynamic filtering system (categories and tags) on the frontend was implemented with vanilla JavaScript to improve user experience without page reloads, while maintaining a functional fallback if JavaScript is disabled.
