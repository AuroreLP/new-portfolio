Memory Game

A simple, accessible and responsive memory matching game with time tracker, flipping cards, match animations, and score tracking. Built with vanilla JavaScript to practice DOM manipulation and game logic.

Features

  • Card flip animations with CSS transitions
  • Real-time timer and move counter
  • Match detection and validation
  • Win condition with victory modal
  • Responsive grid layout
  • Restart game functionality

Development Specs

  • Frontend: HTML5, CSS3, JavaScript (ES6)
  • Features: DOM Manipulation, Event Handling
  • Design: Responsive Grid, Flexbox
  • Performance: Optimized animations
  • Accessibility: Semantic HTML, ARIA labels
  • Deployed with Github pages

Desktop View

The Challenge

The main challenge of this project was implementing a robust card-matching algorithm that could handle various edge cases. I needed to prevent users from flipping more than two cards at once, ensure smooth animations, and manage game state effectively.

I solved this by implementing a state management system using JavaScript closures and event delegation. The timer function was particularly tricky—I had to ensure it paused during card animations and could be reset cleanly without memory leaks.

Another interesting challenge was creating a shuffle algorithm that guaranteed a solvable game every time. I used the Fisher-Yates algorithm to randomize card positions while maintaining pairs.
